Token Refresh & Rotation
POST /refresh with automatic token rotation for security
0/0 ▾
Implement POST /refresh — silent token rotation
src/controllers/auth.controller.js → refreshToken():
  1. Read req.cookies.refresh_token — 401 if missing
  2. Verify JWT: jwt.verify(token, JWT_REFRESH_SECRET) — 401 if invalid/expired
  3. Hash the token and find matching Session doc
  4. If session not found: possible token reuse attack → Session.deleteMany({ user_id: payload.sub }) (revoke ALL sessions) → 401: 'Security event: all sessions revoked'
  5. Verify session not expired (belt-and-suspenders check)
  6. Delete old session doc
  7. Create new session: createSession(userId, req, session.remember_device)
  8. Return 200 { access_token } + set new refresh cookie
🔒 Token rotation means each refresh token can only be used ONCE. If a valid refresh token is presented but no session exists — an attacker stole and used it first. Revoke everything immediately.
