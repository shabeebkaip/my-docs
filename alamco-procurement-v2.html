BOQ File Upload
Entry Point · Step 1
What happens inside
1
File Ingestion & Validation
System accepts .xlsx, .xls, .pdf, .csv, .docx. Validates readability and structure before any processing — corrupted files rejected immediately.
2
Format Detection & Routing
Auto-detects file type. Routes to structured Excel parser, PDF/OCR extractor, or AI pipeline for non-standard legacy layouts.
3
Project Metadata Capture
User attaches project name, reference, client, site location, target delivery date — this context travels with every record downstream.
4
Encrypted Transfer & Job Queue
File + metadata dispatched over encrypted connection to AI Agent. A unique job ID is created. No data stored in third-party systems.
Supported formats
✓
Excel (.xlsx / .xls)
Merged cells, multi-sheet workbooks, hidden rows — all handled.
✓
PDF (native & scanned)
Native text extraction + OCR for scanned BOQs.
✓
Word / CSV
Unstructured Word specs and flat CSV exports both supported.
✓
Non-standard layouts
AI pipeline handles irregular column structures and mixed-language BOQs.
Output →
Validated, format-detected file securely queued for AI processing — with project context and unique job ID attached.
Auto format detection Encrypted transfer Audit job ID created No third-party storage
🤖
AI Procurement Agent
Intelligence Core · Step 2
Extraction & reasoning
1
Division & Section Recognition
AI maps the BOQ hierarchy: Division → Section → Line Item. Handles irregular layouts, mixed languages, and missing headers through contextual reasoning.
2
Line Item Extraction
Extracts description, quantity, unit (m², m, nr, kg…), division tag, and spec codes. Continuation rows merged; subtotals excluded automatically.
3
Quantity Validation & Unit Normalisation
Checks arithmetic vs BOQ subtotals. Normalises units (SQM → m², LM → m). Outlier quantities trigger automatic alerts regardless of confidence score.
Quality control
4
Catalogue Lookup (Gold flow)
Each item matched to Material Template Library in parallel with extraction — not after. Semantic matching, not keyword search.
5
Confidence Scoring
≥90% → auto-proceed · 70–89% → flagged amber for review · <70% → held, officer notified. Prevents incorrect PRs entering Odoo.
6
Structured Output Generation
Final validated procurement list in structured JSON — ready for Odoo push. Every item has SKU, vendor, qty, unit, project tag, and confidence score.
Output →
Fully structured, confidence-scored procurement list — every line item extracted, classified, catalogue-matched, quantities validated, units normalised.
Claude Opus 4 — AI engine Semantic catalogue matching Confidence scoring 0–100% Division hierarchy mapping
📚
Material Template Library
Catalogue Intelligence · Reference Node
What the library holds
·
Per-product data
Internal SKU, category, preferred brand(s), approved suppliers, current unit cost (from last PO), lead time (days), minimum order qty, and applicable BS/EN specifications.
·
Version control
Every change is timestamped and attributed. Full history of cost and spec changes queryable at any time.
·
Bulk import
Upload a new price list or product catalogue — AI merges it, flags conflicts, never blind-overwrites existing entries.
How matching works
1
Vector semantic search
"Ceramic floor tile 600×600 anti-slip", "anti-skid tile 60cm sq.", "porcelain floor 600mm" → all resolve to the same SKU despite different phrasing.
2
Unmatched item handling
No match found → structured entry created, flagged for manual review. Correction is saved — library learns for future projects.
3
Self-improvement loop
Every manual correction improves future match rates. After 3–5 projects the library knows Alamco's exact procurement vocabulary.
Output →
Each BOQ line enriched with: matched SKU, preferred supplier, unit cost, lead time, and min order qty — feeding directly into the PR created in Odoo.
Vector semantic search Version-controlled catalogue Learns from corrections Bulk import supported
🏭
Odoo 19 ERP — Purchase Module
ERP Integration · Step 3
How records are created
1
API Connection & Auth
Connects via Odoo JSON-RPC API with OAuth2. Persistent connection — no re-auth per job. Scoped to Purchase Requisitions write + Products/Vendors read only.
2
Purchase Requisition Creation
For every approved item: product, qty, UoM, delivery date (timeline + lead time), preferred vendor, estimated price, project ref, division tag. All fields populated — no partial records.
3
Product Verification
Product verified against Odoo master before creating the PR. New products → draft record created and flagged, not blocked. Prevents orphan requisitions.
Audit & vendor logic
4
Vendor Assignment
Preferred vendor from library cross-checked against Odoo vendor list. Falls back gracefully — never blocks record creation. Configurable per-category logic (e.g. always Vendor A for tiles above SAR 50K).
5
Full Audit Trail in Odoo
Original BOQ attached to the project record. Each PR chatter note contains: AI timestamp, confidence score, matched library entry, Codeox job ID. Full traceability from BOQ line to PO — inside Odoo.
Output →
Live Purchase Requisition records in Odoo 19 — product-matched, vendor-assigned, quantity-correct, project-tagged. Ready to convert to RFQs or POs.
Odoo 19 JSON-RPC API OAuth2 scoped auth Zero manual entry Full audit trail in Odoo
📡
Status & Delivery Tracker
Real-time Monitoring · Step 4
Monitoring pipeline
1
Continuous Odoo Sync
Polls Odoo Purchase module every 15 minutes. Status changes (Draft → Confirmed → Order Placed → In Transit → Received) auto-detected and recorded in the tracker timeline.
2
Delivery Date Intelligence
Expected delivery = order date + vendor lead time (from library). Missed dates auto-flag the item at-risk → notifies procurement officer → escalates to PM after 24h.
3
5-Tier Alert System
Fires on: (1) PR in Draft >48h · (2) PO not placed within 3 days of approval · (3) Delivery due in 5 days, not shipped · (4) Delivery overdue · (5) Quantity variance on receipt. Email + WhatsApp.
Visibility & feedback
4
Live Dashboard
All active projects with procurement health (Green/Amber/Red), per-project item status, vendor on-time rate, spend vs BOQ estimate, outstanding PRs by age. Auto weekly PDF report to PMs.
5
Feedback Loop → Library
Vendor delivery performance feeds back into the Material Library — lead times auto-adjusted, cost estimates flagged for update. System gets more accurate every project cycle.
Output →
Continuous procurement visibility — delays surfaced before they hit site, vendor performance captured, feedback loop improves every future BOQ cycle.
15-min sync cycle 5-tier alert system Email + WhatsApp alerts Self-improving lead times
