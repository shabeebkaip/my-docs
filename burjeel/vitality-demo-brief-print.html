Unanswered Questions — Confirm with Client Before Demo
Q4 — How do patients currently request a home visit?
Not answered. Demo uses WhatsApp as intake. Ask: Do patients currently call the clinic, use an app, or a website?
Q7 — How should therapists be assigned?
Demo default: nearest therapist + specialization match + availability. Confirm if gender preference, preferred therapist, or other criteria apply.
Q8 — Wait time before notifying next therapist?
Demo default: 10 minutes. Confirm preferred buffer window.
Q9 — Types of files patients might send?
Demo default: PDF (prescriptions, reports, referral letters), JPEG/PNG (wound photos, X-rays).
Q10 — What should AI do with patient files?
Demo default: (1) Acknowledge receipt to patient. (2) Attach to booking record. (3) Notify coordinator that a document is pending review.
Q11 — How far in advance should reminders be sent?
Demo default: 24 hours before + 2 hours before the appointment.
Q12 — How should reminders be sent?
Demo default: WhatsApp message (primary). Optional voice call (secondary, per Respondent 1). Confirm if SMS is also required.
Q14 — Existing HIS / scheduling system?
Not answered. Demo uses dummy data only (confirmed by both). Ask: Is there an existing system for the production integration?

Suggested Tech Stack for Demo
Component Suggested Option Notes
WhatsApp Integration Meta Cloud API or Twilio WhatsApp Sandbox Twilio sandbox is fastest to set up for demo purposes
AI / NLP Engine Claude API (Anthropic) or GPT-4o For understanding messages, detecting language, generating replies
Language Detection Handled by AI model automatically Detect Arabic/English from incoming text; reply in same language
Backend / Orchestration Node.js (Express) or Python (FastAPI) Webhook receiver, booking logic, therapist assignment algorithm
Therapist Assignment Haversine formula or Google Maps Distance Matrix Calculate nearest therapist from patient location using dummy lat/lng
Document Processing Store file + acknowledge patient (demo only) Full OCR/extraction is a production feature — not required for demo
Reminders / Scheduler node-cron or APScheduler (Python) Trigger WhatsApp reminder messages at configured intervals
Voice Call (optional) Twilio Voice API For UC-9 — nice to have
Data Store (demo) JSON file or SQLite Keeps demo setup simple — no production database needed
Demo Dashboard (optional) Simple web UI — live bookings, assignments, escalation log Useful during the demo presentation to show the coordinator's view

Suggested Demo Script — Walkthrough Order
Scene What to Show Use Cases
Scene 1 Patient messages in English: "I need a physiotherapist at home tomorrow morning." AI collects location, date/time. Assigns nearest therapist. Therapist accepts. Booking confirmed via WhatsApp. UC-1, UC-2
