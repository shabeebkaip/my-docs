Section 01
Deployment Overview
A unified, always-on patient coordination platform spanning voice and WhatsApp, integrated directly with Vitality's Nixpend HMS and deployed on Saudi-region cloud infrastructure.
AIWA
WhatsApp omnichannel — inbound patient conversations and outbound marketing campaigns
AIKA
24/7 voice receptionist — inbound calls, therapist confirmations, reminders, outbound dialling
Dashboard
Unified Dashboard — coordinators see conversations, bookings, campaigns, and escalations in one place
Note on AIWA Number

We strongly recommend a dedicated WhatsApp Business number for the AIWA AI agents, separate from any human-operated Vitality numbers. AI agents perform best when they have sole ownership of a number, avoiding conflicts with parallel human-led conversations and ensuring a consistent patient experience.

AIKA – AI Calling / Voice Agent   |   AIWA – AI Chat-based / WhatsApp Agent

Section 02
AIWA — WhatsApp Agent
Patient-Facing
  • 24/7 inbound conversations in Arabic and English within the same chat
  • Conversational booking — service type, preferred date and time, location
  • Location via patient record or Google Maps link for therapist matching
  • Document handling — images (JPEG, PNG) and PDFs (prescriptions, referrals, wound photos, reports)
  • Booking confirmation with therapist name, service, date, time

Outbound Marketing
  • Campaign module for WhatsApp broadcasts — promotions, seasonal offers, re-engagement
  • Template message management with Meta approval workflow
  • Patient segmentation by service type, district, last visit, and visit history
  • Campaign scheduling and response capture
Operational
  • Multi-number support across service lines or geographies
  • Handoff to human coordinator with full conversation context
  • Escalation on urgent language, no availability, or explicit human request
  • Live write-back to Unified Dashboard on every booking milestone

Section 05
Integrations
Nixpend HMS (ERP)
Therapist availability, district-based matching, booking write-back, and patient record sync. System of record.
SMS Gateway
Transactional patient messaging where WhatsApp is unavailable or as a fallback channel.
Payment Gateway
Payment link generation, collection, and reconciliation against bookings.
Google Maps
Patient location capture and resolution for therapist district matching.
Email
Therapist booking notifications with patient documents attached; coordinator escalation alerts.
Integration Assumptions

Nixpend: Pricing assumes REST API access with standard endpoints for availability, booking, and patient records. API documentation not yet received. If non-standard integration is required (SOAP, direct DB, file-based), cost will be revised accordingly.

SMS & Payment Gateways: Integrations are included in pricing. Specific providers are not yet confirmed — final cost may vary based on each provider's API complexity once selected.

06
Section 06
Security & Data Handling
  • Hosting — Saudi-region cloud (AWS Bahrain / KSA, Alibaba Cloud KSA, or other approved Saudi-region providers), compliant with local data residency expectations.
  • Encryption — TLS 1.2+ in transit for all traffic; AES-256 encryption at rest for databases, call recordings, and stored documents.
  • Data Residency — All patient data (conversations, recordings, documents, bookings) remains within the Saudi region.
  • Credential Security — API keys, SIP credentials, and Nixpend tokens stored in encrypted secrets management.
  • Audit Trail — All booking actions, escalations, and data access are logged for operational review.
Compliance Note

Formal compliance certifications (PDPL audit, ISO 27001, SOC 2) are not in scope for this engagement. If Vitality's IT or legal team later requires formal certification, it can be scoped as an add-on.
