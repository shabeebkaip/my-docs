Executive Summary

Platform

Built with Mocha (no-code SPA builder). The site renders entirely via JavaScript — the HTML shell delivered to crawlers is nearly empty, making traditional SEO extremely difficult.

Root Problem

Client-Side Rendering (CSR) is the single biggest SEO risk. Googlebot may not execute JavaScript reliably, meaning the entire page content could be invisible to search engines.

Biggest Quick Win

Fix the og:url mismatch (points to a Mocha staging domain), add a proper meta description, and create robots.txt + sitemap — all achievable within one working day.

Long-Term Priority

Migrate to a platform that supports Server-Side Rendering (SSR) or Static Site Generation (SSG) — such as Next.js, Astro, or WordPress — to unlock full indexability.

Critical Issues
Critical
Client-Side Only Rendering (JavaScript SPA)
The entire page content is invisible to search engine crawlers in static HTML

The website is built with Mocha — a React-based single-page application. The raw HTML returned by the server is essentially an empty shell with <div id="root"></div>. All content — headings, services, contact info, text — is injected at runtime by JavaScript. While Googlebot can sometimes render JavaScript, it is unreliable, delayed, and may miss dynamically loaded content. This is the most severe SEO issue on the site.

Current HTML (what crawlers see)
<body> <div id="root"></div> <!-- Empty. No content visible --> <script src="/assets/index-uh1mQkVb.js"></script> </body>

Critical
Missing robots.txt
https://physiotrio.sa/robots.txt returns 404

A missing robots.txt file means search engine crawlers have no guidance on how to crawl the site. Without it, crawlers may waste crawl budget on irrelevant pages, and Google Search Console will flag this as an error. This file is mandatory for any production website.

Create at: /robots.txt
User-agent: * Allow: / Sitemap: https://physiotrio.sa/sitemap.xml
Critical
Missing XML Sitemap
https://physiotrio.sa/sitemap.xml returns 404

Without an XML sitemap, search engines must discover pages by crawling links alone. For a JavaScript-rendered site, this is especially dangerous — crawlers may never find deep pages. A sitemap tells Google exactly what pages exist and when they were last updated.

Minimum sitemap to create at: /sitemap.xml
<?xml version="1.0" encoding="UTF-8"?> <urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9"> <url> <loc>https://physiotrio.sa/</loc> <lastmod>2026-04-03</lastmod> <changefreq>weekly</changefreq> <priority>1.0</priority> </url> <!-- Add all sub-pages here --> </urlset>
Critical
No Schema / Structured Data Markup
Zero JSON-LD schema detected — no rich results possible

There is no structured data (JSON-LD schema) on any page. For a physiotherapy clinic, this is a major missed opportunity. Schema markup enables Google rich results such as business name, star ratings, services, opening hours, and location in search results — dramatically improving click-through rates.

Recommended: LocalBusiness + MedicalBusiness Schema
<script type="application/ld+json"> { "@context": "https://schema.org", "@type": "MedicalBusiness", "name": "PhysioTrio", "url": "https://physiotrio.sa", "logo": "https://physiotrio.sa/logo.png", "telephone": "+966-XX-XXXXXXXX", "address": { "@type": "PostalAddress", "streetAddress": "[Street Address]", "addressLocality": "Riyadh", "addressCountry": "SA" }, "openingHours": ["Mo-Fr 08:00-20:00", "Sa 09:00-17:00"], "medicalSpecialty": "Physiotherapy", "sameAs": [ "https://www.instagram.com/physiotrio", "https://www.linkedin.com/company/physiotrio" ] } </script>

High
Missing <meta name="description">
No standard HTML meta description — only an OG description exists

While an og:description is present, there is no standard <meta name="description"> tag. Google uses the meta description to generate the snippet shown in search results. Without it, Google will auto-generate a description — often a poor choice — reducing click-through rates by an estimated 30–40%.

Add to <head>
<meta name="description" content="PhysioTrio offers expert physiotherapy and rehabilitation services across Saudi Arabia. Part of the Burjeel Physio Network — book your appointment today." /> <!-- Keep between 140–160 characters -->
High
Missing Canonical Tag
No <link rel="canonical"> — duplicate content risk

Without a canonical tag, Google may index multiple versions of the same page (with/without trailing slash, HTTP vs HTTPS, www vs non-www). This splits link equity and can cause ranking dilution. A canonical tag explicitly declares the preferred URL version.

Add to <head>
<link rel="canonical" href="https://physiotrio.sa/" />
High
Wrong Favicon & Social Image (Mocha Defaults)
Favicon and OG image point to Mocha CDN, not PhysioTrio brand assets

Both the favicon (getmocha.com/favicon.ico) and the Open Graph social preview image (static.getmocha.com/og.png) are Mocha branding assets. When this page is shared on WhatsApp, LinkedIn, or Twitter, it will show the Mocha logo — not PhysioTrio's. This undermines brand recognition and trust.

Fix
<link rel="shortcut icon" href="/favicon.ico" /> <meta property="og:image" content="https://physiotrio.sa/og-image.jpg" /> <!-- Recommended OG image size: 1200x630px -->
