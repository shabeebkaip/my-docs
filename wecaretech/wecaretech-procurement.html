Key Pain Points — ICT Procurement Context
📧 Email BOQ Arrives — Nothing Happens Automatically
Clients send BOQ or project requirements via email. A procurement officer must manually read, interpret, and re-enter data into spreadsheets. Items like Cat6A UTP Cables, Cisco Catalyst switches, Hikvision PTZ cameras require different vendor channels — none mapped automatically.
🔌 Multi-Category Complexity: No Unified System
A single data center project can require structured cabling (R&M, Panduit), active networking (Cisco, Juniper), UPS/PDU (Eaton, APC, Schneider), surveillance (Hikvision, Hanwha), and access control (Nitgen) — each with separate vendors, lead times, and approval paths.
📊 Zero Visibility Across Concurrent Projects
Management has no real-time view of procurement status across active projects. Items stuck at approval, long-lead hardware not yet ordered, and budget overruns are discovered after the fact.
Critical Risk: In ICT and data center projects, missing procurement windows for long-lead items — server hardware, UPS systems, fiber optic trunk cables — can delay project delivery by 4–12 weeks, threatening client SLAs and contract penalties.

AI Engine
The AI Brain — How It Works
Five specialized AI components, each responsible for one step of the procurement intelligence pipeline. All powered by Claude Opus 4 and GPT-4.1.
05
1
BOQ Document Parser
Claude Opus 4 + Python openpyxl
Core
Accepts Excel (.xlsx), PDF, and Word BOQ documents. For Excel: structured Python parser with hierarchical pattern recognition. For PDF/Word: Claude API processes the document with a custom extraction prompt tuned to ICT BOQ formats.
Input: "Cat6A UTP Cable — 4PR 23AWG — 305m box — Qty: 50 boxes"
→ Category: Structured Cabling / Copper
→ Product: UTP Cat6A · Qty: 50 × 305m
→ Preferred Brands: R&M, Panduit, Datwyler
→ Lead Time Flag: Standard (7–14 days)
2
Material Requirements Expander
Claude Opus 4 + ICT Material Template Library
Core
For each BOQ line item, AI expands to full material breakdown — primaries, accessories, consumables. Uses pre-built templates for 60+ ICT scope categories (data center racks, fiber optic runs, camera installations, UPS systems…). LLM fallback for novel items.
BOQ Item: "Install 48-port data cabinet"
→ 1× 42U Open Frame Rack
→ 2× 1U Patch Panel 24-port Cat6A
→ 1× Horizontal Cable Manager
→ 4× D-Ring Cable Guides
→ 1× PDU 16A 8-outlet
→ Cable ties, labels, earthing kit
3
Intelligent Vendor Matcher
PostgreSQL + Pgvector semantic search
Core
Maintains WeCare Tech's full vendor database with vector embeddings of product descriptions. Semantic search matches extracted items to the best 3 vendors per category. Ranks by price history, lead time, and vendor performance score.
Cisco · Juniper · HPE Eaton · APC · Schneider · Vertiv Hikvision · Hanwha · Bosch R&M · Panduit · Datwyler · Legrand Nitgen · HID · Honeywell
4
Smart Procurement Packager
Rule Engine + Claude reasoning
Core
Groups extracted materials into logical procurement packages by scope and vendor category. Auto-assigns release priority based on lead time and project timeline. Generates prerequisite checklist (client approvals, shop drawings, technical submittals) before PO can be raised.
PKG-CAB-01 Structured Cabling · 12 line items · Panduit / R&M
PKG-NET-01 Active Networking · 8 items · Cisco · Long-Lead 6wk
PKG-UPS-01 Power Systems · 4 items · Eaton · Approval Required
PKG-CAM-01 CCTV & Surveillance · 9 items · Hikvision
🔄
AI Learning Loop — The System Gets Smarter With Every Project Future Intelligence
When a procurement officer edits an AI-generated material list or chooses a different vendor, the system records this feedback and updates its templates. Over time, the AI learns WeCare Tech's exact preferences — specific cable grades preferred, which UPS brand performs best on which project type, fastest vendors for critical deliveries. This is a continuously improving system, not a static tool.

Step-by-Step Automated Flow
#StepSystem ActionOutputTime
1 Email Arrives Email watcher detects new message to procurement inbox. Identifies sender type (client, consultant, vendor). Trigger fired Instant
2 Attachment Classification AI classifies attachment: BOQ, RFP, quotation request, or general enquiry. Non-procurement emails ignored or routed to CRM. Document type confirmed <10s
3 BOQ Extraction Claude Opus 4 extracts all line items: item codes, descriptions, quantities, units, scope categories. Confidence score assigned per item. Structured item list <45s
4 Project Auto-Created New project record created with client name, reference number, BOQ total, and extracted scope summary. Project in system <5s
5 Material Expansion Each BOQ line expanded to full material breakdown using template library + LLM. Full Bill of Materials generated per project scope. Complete BOM <90s
6 Vendor Matching Each material matched to top-3 preferred vendors from WeCare Tech's approved supplier list. Historical prices and lead times attached. Vendor shortlist per item <20s
7 Procurement Packages Materials grouped into procurement packages by scope and vendor. Long-lead items flagged. Release priority set based on project timeline. Packages with priorities <10s
8 PR Auto-Draft Purchase requisitions auto-drafted per package. Budget estimate computed from historical prices. Approval workflow assigned based on value thresholds. Draft PRs ready <15s
9 Manager Notified Email + WhatsApp notification sent to procurement manager with summary. Dashboard alert triggered. Approval workflow starts Instant
10 One-Click Approve Manager reviews AI-structured PRs in dashboard. Approves, edits, or escalates. Audit trail recorded for all actions. Approved PRs → PO Human step
