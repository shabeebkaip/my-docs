Why WordPress is the Wrong Platform for Jethur
The Case for Migration

During our initial discovery meeting, the WordPress-to-Next.js migration was flagged as a strategic priority. Below is the structured case for why continuing on WordPress poses both technical and business risks for a company in Jethur's position.

2.1 · The Security Argument
A Compliance Brand on the World's Most-Hacked CMS
WordPress powers approximately 43% of the internet, which also makes it the most targeted CMS by attackers worldwide. For a company whose core product is helping clients achieve NCA ECC compliance and manage cybersecurity risk, this creates a direct contradiction. A GRC vendor running on the most-exploited platform is a credibility liability.
2.2 · The Credibility Argument
Enterprise Buyers Audit Your Security Posture
Your target audience — CISOs, compliance officers, risk managers, and C-suite executives across Saudi government and private sectors — are sophisticated buyers. Enterprise procurement processes increasingly include vendor cybersecurity assessments. A Next.js application on hardened infrastructure passes these assessments far more convincingly than a CMS-dependent site with plugin dependencies.
2.3 · The Performance Argument
Native Performance vs. Expensive Optimisation
WordPress performance degrades significantly without expensive optimisation layers — caching plugins, CDN configurations, database tuning. Next.js 16 delivers static generation and server-side rendering natively, resulting in near-instant page loads that improve both user experience and search engine ranking, with zero additional overhead.

Project Scope
5 Workstreams

The following scope covers the full end-to-end revamp of jethur.com across five workstreams: Discovery & Strategy, Design (Figma), Development (Next.js 16), Custom CMS Build, and Launch & Handover.

3.1 — Discovery & Strategy
  • Stakeholder alignment session to define sitemap, messaging hierarchy, and content priorities
  • Full audit of existing 22-page WordPress site — content inventory, SEO baseline, traffic analysis
  • Competitor and benchmarking analysis across GRC/cybersecurity platforms in the GCC region
  • Definition of target audience personas — CISOs, Compliance Officers, Risk Managers, C-suite
  • Information architecture design — revised sitemap and page hierarchy
  • SEO strategy document — keyword mapping per page, redirect plan for existing URLs
3.2 — UX & Design (Figma-First)

All design work will be completed in Figma before a single line of code is written. This ensures Jethur has full visibility and sign-off on the experience before development begins.

  • Design System — typography scale, color palette, spacing system, component library
  • Low-fidelity wireframes for all key page templates (desktop and mobile)
  • High-fidelity UI designs for all page templates — pixel-perfect, production-ready
  • Interactive prototype in Figma for stakeholder review and client presentations
  • Responsive design specifications — Desktop (1440px), Tablet (768px), Mobile (375px)
  • Accessibility review against WCAG 2.1 AA standards
  • Micro-interaction and animation specifications
3.3 — Frontend Development — Next.js 16

The site will be built on Next.js 16 using the App Router architecture, deployed on Jethur's own server infrastructure or a recommended Saudi Arabia-based cloud environment — ensuring data residency, sovereignty, and latency performance for the Gulf market.

  • Next.js 16 App Router — server components, static generation, and ISR (Incremental Static Regeneration)
  • TypeScript — full type safety across the codebase
  • Tailwind CSS — utility-first styling, consistent with the Figma design system
  • Component-driven architecture — reusable, documented React components
  • Framer Motion — smooth, professional page transitions and scroll animations
  • Full SEO implementation — meta tags, structured data (JSON-LD), Open Graph, sitemap.xml, robots.txt
  • Performance optimisation — Core Web Vitals targeting 95+ on Lighthouse
  • Security headers — CSP, HSTS, X-Frame-Options, CORS policies configured at deployment
  • Contact forms with spam protection (reCAPTCHA v3 or Cloudflare Turnstile)
  • Analytics integration — Google Analytics 4 or privacy-first alternative (Plausible)
3.4 — Custom Content Management System (Built in Next.js)

Rather than relying on a third-party headless CMS, a fully custom admin panel will be built directly within the Next.js application. This gives Jethur complete ownership of the CMS, no recurring third-party subscription costs, and a content management experience tailored exactly to the team's workflow — without any external platform dependency.

  • Custom admin panel built within the Next.js app — same codebase, no external service
  • Blog / Insights management — create, edit, publish articles with rich text, images, and tags
  • Custom page builder — create new landing pages using pre-built block components without code
  • Team & leadership profiles management
  • Case studies and client success stories management
  • Standards & Regulations library — NCA ECC, SAMA, PDPL, NIST, ISO 27001, etc.
  • Product feature pages management
  • User roles and permissions — editor, publisher, admin
  • Media asset management — image upload, optimisation, and delivery
  • Arabic language content support — RTL-ready architecture
  • No third-party CMS subscription — fully owned by Jethur

4.2 — Deployment & Infrastructure

The application will be deployed on Jethur's own server infrastructure or a recommended Saudi Arabia-based cloud environment — such as AWS Middle East (Riyadh) region or STC Cloud — ensuring data residency within the Kingdom, compliance with NCA ECC data localisation requirements, and optimal latency for Gulf users. The deployment option will be confirmed during the discovery phase based on Jethur's existing infrastructure and preferences.

4.3 — Security Architecture

The following security measures are implemented as standard — not add-ons. These are non-negotiable for a cybersecurity brand.

HTTP Security Headers — CSP, X-Frame-Options, HSTS, Referrer-Policy
No exposed admin panels or backend endpoints on the public domain
All API routes secured with server-side authentication
Cloudflare WAF protecting against OWASP Top 10
Dependency audit pipeline — automated CVE scanning on every code commit
HTTPS-only with automatic certificate renewal (TLS 1.3)
No WordPress, no plugins, no shared execution environment
Environment variables isolated — no secrets in the codebase
Privacy-compliant analytics — no PII shared with third parties without consent
