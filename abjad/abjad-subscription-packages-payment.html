Teacher Subscription — Ranking Algorithm

1.1 Problem Statement

When multiple teachers subscribe to gain top visibility, a simple "pay first, rank first" approach creates unfairness, stale listings, and poor experience for schools. The platform requires a smart, multi-factor ranking system that rewards profile quality, genuine activity, and verified credentials — not just payment order.

Key Challenge: Without a dynamic ranking mechanism, early subscribers permanently monopolise top positions, discouraging new teacher sign-ups and reducing result quality for schools.

1.2 Recommended Approach: Weighted Dynamic Ranking Score (WDRS)

Subscribed teachers are placed in a Premium Pool and ranked among themselves using the Weighted Dynamic Ranking Score (WDRS). Non-subscribed teachers are ranked separately below the premium pool.

Best Option: Hybrid Quality + Subscription Ranking — combines subscription investment with genuine teacher quality signals, ensuring schools always see the most relevant candidates first.
WDRS = (Profile Score × 30%) + (Subscription Tier × 20%) + (Activity Score × 20%) + (Response Rate × 15%) + (Verification Score × 15%)
Maximum possible score: 100 points

Scoring Breakdown:

Factor Weight How It Is Calculated Max Points
Profile Completeness 30% % of profile fields filled — photo, credentials, certifications, CV, bio, all sections 30
Subscription Tier 20% Annual = 20 pts  |  6-Month = 14 pts  |  Monthly = 8 pts 20
Activity Score 20% Last login recency + response rate to interview invitations 20
Response Rate 15% % of school inquiries / interview requests responded to within 48 hours 15
Verification Level 15% Admin-verified documents: Degree +5 pts  |  Certifications +5 pts  |  ID +5 pts 15
Tie-Breaking Rule: When two teachers have an equal WDRS, the one with the earlier subscription start date ranks higher.
Anti-Monopoly Rotation: Subscribed teachers in the top positions rotate every 24 hours among those within equal score bands (±5 points of each other) to ensure fair exposure and prevent permanent position lock-in.

1.4 What Subscribed Teachers Get (Beyond Ranking)

  • "Featured" badge displayed on profile card in all search results
  • Profile appears in school search results before all non-subscribed teachers
  • Receive job match notifications faster — real-time vs. daily digest for free users
  • Profile shown in "Recommended Candidates" section visible to all schools
  • Highlighted profile border in search results for visual distinction

1.5 Admin Control

  • Admin can adjust WDRS weights from the Admin Panel (Feature 4.7.4) without code changes
  • Admin can manually review if a subscribed teacher is gaming the system or inflating activity metrics
  • Suspicious activity flag removes premium visibility boost pending administrative review

  • Monthly Cancellation: Effective at end of current billing month

Payment Flow & ZATCA Compliance

3.1 What is ZATCA E-Invoicing?

The Zakat, Tax and Customs Authority (ZATCA) — هيئة الزكاة والضريبة والجمارك — mandates e-invoicing (Fatoorah / فاتورة) for all VAT-registered businesses in Saudi Arabia.

Phase 1 — Generation
Generate and store e-invoices in ZATCA-approved XML format with cryptographic QR codes for all transactions.
Phase 2 — Integration
Real-time invoice reporting to ZATCA via API using Cryptographic Stamp Identifier (CSID) for B2B transactions.
VAT Rate
15% on all taxable transactions — hardcoded to avoid calculation errors.
Invoice Archiving
All invoices must be stored and accessible for a minimum of 6 years from invoice date — non-negotiable.

3.2 Invoice Types Required

Transaction Type Customer Type Invoice Type Requirements
Subscription Payment School (B2B) Standard Tax Invoice Buyer VAT number, XML format, QR code, UUID
Teacher Subscription Teacher (B2C) Simplified Tax Invoice Seller details, QR code, UUID — no buyer VAT required

Best Option: Moyasar (Primary) + Tamara (BNPL)

3.4 Complete Payment Flow — School Subscription

1
Package Selection

School selects Bronze / Silver / Gold and billing period (Monthly / 6-Month / Annual) from the pricing page.

2
Cart Summary

System displays: package name and features, base price (SAR), VAT amount (15%), total amount (SAR), billing start and end dates.

3
VAT Registration Check

System prompts: "Do you have a VAT Registration Number?"

YES — Collect VAT number → Generate Standard Tax Invoice (B2B)
NO — Proceed → Generate Simplified Tax Invoice
4
Payment Method Selection
Mada Apple Pay STCPay Tamara (BNPL) Visa / Mastercard
5
OTP / 3DS Verification
Mada / STCPay: OTP via SMS
Visa / Mastercard: 3D Secure redirect
Apple Pay: Face ID / Touch ID
Tamara: Tamara app approval
6
Payment Processing (Moyasar Gateway)
Real-time authorisation sent to gateway
Success → Proceed to Step 7
Failure → Show error, offer retry or alternative payment method
7
E-Invoice Generation (ZATCA Compliant)

System automatically generates invoice containing:

UUID (unique invoice identifier, system generated)
Invoice date — Gregorian + Hijri (ميلادي + هجري)
Seller details: Abjad company name, VAT number, CR number, address
Buyer details: school name, VAT number (B2B), address
Line items: package name, period, base amount
VAT amount (15%) + Total amount inclusive of VAT
QR code (ZATCA cryptographic stamp) + Digital signature (CSID for Phase 2)
8
Invoice Distribution
PDF invoice emailed to school billing email address
Invoice available in school dashboard (Payment History)
Invoice stored in Abjad system for 6 years (ZATCA requirement)
Phase 2: Real-time XML submission to ZATCA portal
9
Subscription Activation
School account upgraded immediately upon payment confirmation
Welcome notification sent to school admin
Billing reminder set for renewal date (7 days before expiry)
Auto-renewal processed if enabled by school

3.7 Refund Policy (ZATCA Compliant)

Admin approval required for any refund, per Feature 3.9.6. Refunds cannot be processed unilaterally by the school.

When a refund is approved, the system must:

  • Generate a Credit Note (debit/credit note is ZATCA-mandated for all refunds)
  • Credit Note references the original invoice UUID for traceability
  • Credit Note contains negative amounts matching the original line items
  • Submitted to ZATCA (Phase 2) identically to the original invoice
  • Refund processed back to original payment method only
  • Processing time: 5–10 business days (Mada/local) or 10–15 days (international cards)

3.9 PDPL Compliance Note

In addition to ZATCA, the Personal Data Protection Law (PDPL) of Saudi Arabia — نظام حماية البيانات الشخصية — applies to all payment data handled by Abjad:

  • Payment data must be stored in Saudi Arabia or countries with equivalent data protection
  • School and teacher financial data must not be shared with third parties without explicit consent
  • Users have the right to request deletion of payment history — subject to the 6-year ZATCA archiving override for invoice records
ZATCA vs PDPL: The mandatory 6-year ZATCA invoice archiving requirement legally overrides individual deletion requests for invoice records. Users can be informed of this limitation transparently.
