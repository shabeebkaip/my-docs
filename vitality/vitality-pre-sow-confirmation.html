Overview
Scope of Work
Following the successful pilot, this document confirms scope for the production deployment and lists the remaining inputs we need to issue the Statement of Work. We will deploy the AI patient-coordination platform, already validated in the Vitality pilot, into production for Vitality Home Care across Riyadh.
Production Deployment Covers
  • AIWA (WhatsApp Agent) — Patient intake, conversational booking in Arabic and English, location and document handling, and patient-facing confirmations and escalations.
  • AIKA (Voice Agent) — Therapist confirmation calls after each booking and patient reminder calls 2 hours before the visit.
  • Nixpend HMS Integration — Therapist availability, district-based matching, and booking write-back, replacing the pilot's dummy dataset.
  • SIP Integration — Integration with Vitality's existing telephony infrastructure for all AIKA outbound calls.
  • Saudi-Region Cloud Deployment — Standard data protection controls including TLS and encryption at rest. Hosting cost borne by Vitality.
  • Coordinator Escalation & Handoff Workflows — For cases beyond AI scope.
Included in Deployment

Build, integration, UAT, go-live, and post-launch support are included. Onboarding to additional entities or geographies will be scoped separately once the initial production deployment is live and running smoothly.

Note to Vitality

This scope reflects what we discussed in the meeting. If you would like to add or modify anything — such as marketing outbound campaigns, inbound receptionist coverage (AIKA/AIWA), additional service lines, or any other use case — please update this section directly and return it. We will revise the Statement of Work accordingly.
